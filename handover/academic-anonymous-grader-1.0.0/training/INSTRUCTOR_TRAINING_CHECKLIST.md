# Academic Anonymous Grader — Instructor Training Checklist

## Version and Status
- **Document Version:** 1.0.0
- **Status:** Ready for Training
- **Last Updated:** 2026-06-16

## Instructions
This checklist should be completed by new Instructors (Graders) during onboarding. Each item should be demonstrated and signed off by a trainer.

## Prerequisites
- [ ] Application is installed and running.
- [ ] Trainer has Administrative or Grader access.
- [ ] Trainee has a Grader account.
- [ ] At least one assessment is assigned to the trainee.

## Training Items

### 1. Login
- [ ] Navigate to the application URL.
- [ ] Enter credentials.
- [ ] Confirm Dashboard loads.
- [ ] Verify only assigned work is visible.
- [ ] Logout.

### 2. Assigned-Work Navigation
- [ ] After login, view the Dashboard.
- [ ] Locate assigned grading tasks.
- [ ] Click on an assigned assessment.
- [ ] Verify that only anonymous codes are visible (no names, emails, or IDs).

### 3. Anonymous-ID Interpretation
- [ ] Understand that anonymous codes (e.g., `ANON-P13-001`) are randomly generated.
- [ ] Understand that you cannot reverse-map an anonymous code to a student identity.
- [ ] Understand that seeking to identify students from anonymous codes is prohibited.

### 4. Grading
- [ ] Open a student submission.
- [ ] Enter a grade.
- [ ] Enter feedback.
- [ ] Save the grade.
- [ ] Verify the grade appears in the grading interface.

### 5. Feedback
- [ ] Enter detailed feedback.
- [ ] Save.
- [ ] Verify feedback is saved.

### 6. Draft Saving
- [ ] Start grading a submission.
- [ ] Enter partial grade and feedback.
- [ ] Save as draft.
- [ ] Return later and verify the draft persists.

### 7. Completion
- [ ] Complete all grading for an assessment.
- [ ] Verify the completion indicator updates.
- [ ] Confirm no further submissions need grading.

### 8. Returned Corrections
- [ ] If an assessment is returned for correction by a Reviewer:
- [ ] View the returned submissions.
- [ ] Adjust grades or feedback as needed.
- [ ] Re-submit completed status.

### 9. Scoped Analytics
- [ ] Navigate to Analytics.
- [ ] View assessment-level analytics.
- [ ] Verify only the instructor's assigned assessments are visible.
- [ ] Verify no identity data appears in analytics.

### 10. Privacy Obligations
- [ ] Understand that student identities must never be sought.
- [ ] Understand that discussing anonymous codes or grades publicly may violate student privacy.
- [ ] Understand that grading workstations should not be left unattended while logged in.

### 11. Prohibited Identity Seeking
- [ ] Explicitly acknowledge that attempting to map anonymous codes to student identities is a policy violation.
- [ ] Report any accidental identity exposure immediately to the Administrator.

### 12. Logout and Workstation Security
- [ ] Logout after each grading session.
- [ ] Lock workstation when stepping away.
- [ ] Never share credentials.
